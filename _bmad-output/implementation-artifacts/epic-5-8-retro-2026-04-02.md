## What Went Well

**ScanCoordinator design held across 3 epics.** Designed once in Story 5.2, reused without modification through all of Epic 7. Zero rework.

**Real database integration tests.** Backup/restore tests (8.1) used actual SQLite files on disk, not mocks. Caught the WAL checkpoint requirement that mocks would have hidden entirely.

**gRPC removal was clean.** Story 5.6 excised all proto/grpc/fixnum dependencies in one pass. Nothing broke. No stories tangled in it afterwards.

**Design decisions made once, not revisited.**
- `internal/client/` package to break circular import (6.3) — decided and done
- Polling over streaming for IntakeProvider (7.2) — accepted tradeoff, 500ms latency acceptable
- `exit(0)` for restore restart (8.1) — MVP pragmatism, no over-engineering

**Strong test discipline throughout.** Every story shipped with comprehensive tests. Stories touching integration (backup, HTTP handlers) used real infrastructure, not mocks.

---

## Challenges

**Resource management gaps were a recurring review pattern.** Unbounded buffers, missing timeouts, socket leaks, and undrained response bodies appeared in Stories 5.1, 5.4, 6.2, 8.1, and 8.2. The review process caught all of them, but the pattern was consistent enough to be structural.

Root cause: no checklist for I/O stories. Implementation focus naturally goes to happy path and error codes — "did I drain the response body on the error branch?" isn't top of mind without a prompt.

**Story 5.5 was the noisiest review.** 8 patches + 8 defers + 6 dismissed items. Mid-story scope expansion (ConnectionProvider → ServerProvider, IntakeScreen rewrite) created review noise. Fuzzy scope correlates with noisy reviews.

---

## Key Insights

1. The review process earned its place — it caught real bugs every time, but most of those bugs followed a predictable I/O resource management pattern that a checklist could prevent.
2. Designing coordination primitives well (ScanCoordinator) pays dividends across multiple epics.
3. MVP tradeoffs (exit(0), no loopback auth, no progress bars) were correct calls — none created problems.
